## Component Inventory

### 1. Upload Button Section
- Button "Subir vídeo para Opus" — appears after moments are ranked and extraction is complete
- States: idle (blue bg), loading (spinner + "Enviando..."), success (green check + disabled), error (red border + error message)
- Progress bar below button during upload showing percentage

### 2. Timestamp Verification Modal
- Modal overlay with confirmation before sending each moment
- Contains: video player preview at timestamp, moment details (start time, duration, transcript snippet)
- Actions: "Confirmar" (sends), "Cancelar" (closes modal)
- The modal appears per-moment when user clicks individual send button

### 3. Moment Send Controls
- Per-moment: small button on each MomentCard to send individually (triggers verification modal)
- "Send All" button: sends all selected moments without individual verification
- Both buttons disabled until video upload to Opus is complete

### 4. Send Status Indicators
- Per-moment: checkmark or spinner after sending
- Job ID display after successful send

---

## Layout Structure

### Phase 4 Flow
1. User processes video → sees ranked moments
2. "Subir vídeo para Opus" button appears in highlights/dashboard section
3. User clicks upload → progress bar shows upload status
4. After upload success: timestamp send buttons become enabled
5. User can:
   - Click individual moment's send button → verification modal → confirm → sent
   - Click "Enviar todos" → all selected moments sent at once (no verification)
6. Success state shows job ID
