<feature>
  <name>Audio Transcription with Faster Whisper</name>
  <files>backend/services/transcription.py, backend/tests/test_transcription.py</files>
  <behavior>
    - transcribe_file(video_path: str, model_size: str = "base") -> dict
    - Input: path to video/audio file
    - Output: {transcript: str, segments: [{id, start, end, text}], words: [{word, start, end, confidence}], duration: float, language: str}
    - segments have word-level timestamps (word_timestamps=True)
    - 60-minute file processes without memory issues
    - Uses 'base' model (~1GB RAM) for good speed/accuracy balance
  </behavior>
  <implementation>
Use faster_whisper library (not openai-whisper).

Implementation from transcription.py reference:
1. Load model: faster_whisper.load_model("base")
2. Transcribe: model.transcribe(file_path, word_timestamps=True)
3. Extract segments with word-level timing
4. Return structured JSON (transcript, segments, words, duration, language)
5. Save to /tmp/clipwise/<video_id>/transcript.json

Adapt from podcli transcription.py but use faster-whisper instead of openai-whisper.
</implementation>
</feature>
